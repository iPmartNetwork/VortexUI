## Webhook (HMAC-SHA256)

**الإعدادات → الإشعارات → Webhook**

استقبال حمولات JSON منظمة لأحداث لوحة التحكم على نقطة النهاية الخاصة بك.

### الإعدادات

| الإعداد | الوصف |
|---------|-------|
| الرابط (URL) | نقطة نهاية الويب هوك الخاصة بك |
| السر | مفتاح توقيع HMAC-SHA256 |
| الأحداث | اختيار الأحداث المراد إرسالها |
| مفعّل | تشغيل/إيقاف |

### التحقق

كل طلب يتضمن ترويسة `X-Signature-256`:

```
X-Signature-256: sha256=<hex_hmac>
```

تحقق بحساب `HMAC-SHA256(request_body, secret)` ومقارنته.

### بنية حمولة الحدث

```json
{
  "event": "user.limited",
  "timestamp": "2025-01-15T10:30:00Z",
  "data": {
    "user_id": "uuid",
    "username": "testuser",
    "data_limit": 53687091200,
    "used_traffic": 53687091200
  }
}
```

### الأحداث المتاحة

| الحدث | المُحفِّز |
|-------|----------|
| `user.created` | إنشاء مستخدم جديد |
| `user.deleted` | حذف مستخدم |
| `user.limited` | وصول المستخدم لحدّ البيانات |
| `user.expired` | انتهاء اشتراك المستخدم |
| `user.expiry_warning` | قبل 3 أيام من الانتهاء |
| `user.enabled` | إعادة تفعيل المستخدم |
| `user.disabled` | تعطيل المستخدم يدوياً |
| `node.offline` | فقدان اتصال العقدة |
| `node.online` | إعادة اتصال العقدة |
| `node.unhealthy` | فشل فحص سلامة العقدة |
| `order.created` | طلب شراء جديد |
| `order.paid` | تأكيد دفع الطلب |
| `backup.completed` | اكتمال النسخ الاحتياطي |
| `admin.login` | حدث تسجيل دخول المسؤول |

## بوت تيليجرام

**الإعدادات → الإشعارات → تيليجرام**

### بوت المسؤول

يرسل الإشعارات إلى محادثة تيليجرام الخاصة بالمسؤول:

| الإعداد | الوصف |
|---------|-------|
| توكن البوت | من [@BotFather](https://t.me/BotFather) |
| معرّف محادثة المسؤول | معرّف محادثتك الشخصية أو الجماعية |
| الأحداث | اختيار أحداث الإشعار |

### البوت الموجّه للمستخدمين

يمكن للمستخدمين التفاعل مع البوت باستخدام توكن اشتراكهم:

| الأمر | الإجراء |
|-------|---------|
| `/start` | ربط الحساب بتوكن الاشتراك |
| `/usage` | عرض استهلاك البيانات الحالي |
| `/renew` | الحصول على رابط التجديد |
| `/status` | فحص حالة الحساب |
| `/help` | عرض الأوامر المتاحة |

### قوالب الإشعارات

تخصيص صيغة الرسالة بمتغيرات القالب:

```
🔔 مستخدم مقيّد
اسم المستخدم: {username}
المستخدم: {used_traffic}
الحدّ: {data_limit}
```

## إشعارات الحصة

**الإعدادات → الإشعارات → تنبيهات الحصة**

تنبيه المستخدمين عند اقترابهم من حدّ البيانات:

| الإعداد | الوصف |
|---------|-------|
| مفعّل | تفعيل إشعارات الحصة |
| العتبات | النسب المئوية للتفعيل (مثل 80%، 90%، 100%) |
| تيليجرام | الإرسال عبر البوت |
| Webhook | الإرسال لرابط الويب هوك |
| فترة التبريد | دقائق بين التنبيهات المتكررة |

---

## تنبيهات حصة الموزّع

**الشريط الجانبي → تنبيهات حصة الموزّع** (مسؤول sudo فقط)

مراقبة اقتراب الموزّعين من حدود حصتهم (حركة/مستخدمين):

| الإعداد | الوصف |
|---------|-------|
| مفعّل | مفتاح عام |
| تيليجرام | الإرسال لبوت لوحة التحكم |
| رابط Webhook | نقطة نهاية خارجية اختيارية |
| العتبات | النسب المئوية (مثل 80، 90، 100) |
| فترة التبريد | دقائق بين التنبيهات |
| التنبيهات الأخيرة | جدول التنبيهات المُفعّلة |

---

## مركز الإشعارات (قائمة الجرس)

أيقونة الجرس في رأس لوحة التحكم تعرض الإشعارات الأخيرة:

- شارة عدد غير المقروءة
- انقر للتوسيع
- كل إشعار يعرض: نوع الحدث، الوصف، الطابع الزمني
- انقر للانتقال للمورد المعني
- وضع علامة مقروء / وضع علامة الكل مقروء

تُحفظ لكل حساب مسؤول.

## أحداث SSE الحيّة

تستخدم لوحة التحكم أحداث مرسلة من الخادم (SSE) لتحديثات الواجهة الفورية:

| البث | المحتوى |
|------|---------|
| `/api/sse/events` | أحداث النظام (حالة العقد، حدود المستخدمين، إلخ) |
| `/api/sse/stats` | إحصائيات حيّة (الاتصالات، عدّادات الحركة) |
| `/api/sse/monitor` | تحديثات الاتصالات النشطة |

مكونات الواجهة تشترك تلقائياً. لا حاجة لإعداد — SSE نشط دائماً.

!!! tip
    أحداث SSE تشغّل المقاييس الحيّة في لوحة المعلومات وصفحة المراقبة وجرس الإشعارات.
    إذا كان الوكيل العكسي يخزّن الاستجابات مؤقتاً، تأكد من السماح ببث SSE (Caddy يتعامل مع هذا افتراضياً).
