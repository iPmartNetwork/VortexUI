# 17. VortexUI v1.2.3 — دليل الميزات الجديدة

!!! info "Version 1.2.3"
    تستعرض هذه الصفحة جميع الميزات المُقدَّمة في VortexUI v1.2.3. يشرح كل قسم وظيفة الميزة وأين توجد في واجهة المستخدم وكيفية إعدادها.

---

## مضيفو الاشتراك (Subscription Hosts)

**الموقع:** الشبكة والعُقد → مضيفو الاشتراك (يمكن الوصول إليها أيضًا من تب **Hosts** الخاص بكل inbound)

تجاوزات المضيف لكل inbound، على نمط Marzban. يغيّر مضيف الاشتراك ما يُعلن عنه الرابط المُولَّد في اشتراك المستخدم — العنوان، SNI، هدر Host، المسار، أمان النقل، والمزيد — **دون** المساس بإعدادات الهسته الحيّة على العُقدة. يتيح لك ذلك وضع inbound واحد خلف عدة نطاقات CDN أو عدة قيم SNI أو عدة منافذ.

### ما يمكنك تجاوزه

| Field | الوصف |
|-------|-------------|
| Remark | اسم العرض للإدخال (يدعم متغيرات القالب) |
| Address | المضيف/IP المُعلن للعملاء (مثل نطاق CDN) |
| Port | تجاوز المنفذ (`0` = توريث منفذ inbound) |
| SNI | اسم خادم TLS |
| Host | هدر `Host` لـ HTTP |
| Path | مسار WS/HTTPUpgrade/gRPC |
| ALPN | قائمة مفصولة بفواصل (مثل `h2,http/1.1`) |
| Fingerprint | بصمة uTLS (مثل `chrome`) |
| Security | `inbound_default` أو `none` أو `tls` أو `reality` |
| Allow insecure | تخطّي التحقق من الشهادة |
| Mux | تفعيل المالتيبلكسينغ |
| Fragment | مواصفات fragment لـ TLS (مثل `1,40-60,30-50`) |
| Priority | ترتيب تصاعدي ضمن inbound |
| Enabled | تشغيل/إيقاف المضيف |

### متغيرات القالب

تُولَّد الحقول النصية لكل مستخدم، لذا يصلح تعريف مضيف واحد للجميع:

| Variable | يتوسّع إلى |
|----------|-----------|
| `{USERNAME}` | اسم مستخدم المستخدم |
| `{SERVER_IP}` | عنوان العُقدة |
| `{SERVER_PORT}` | المنفذ المُعلن |
| `{PROTOCOL}` | بروتوكول inbound (vless، vmess، …) |
| `{NETWORK}` | النقل (tcp، ws، grpc، …) |
| `{SECURITY}` | أمان النقل (tls، reality، none) |
| `{REMARK}` | الـ remark المُعدّ |

### الإعداد

1. افتح inbound وانتقل إلى تب **Hosts** (أو اذهب إلى **مضيفو الاشتراك**)
2. انقر على **Add Host**
3. املأ حقول التجاوز التي تحتاجها — اترك الباقي فارغًا ليُورَّث من inbound
4. اسحب الصفوف (أو استخدم **Reorder**) لضبط الأولوية — الأولوية الأقل تُولَّد أولًا
5. احفظ. تعكس عملية جلب الاشتراك التالية المضيفين الجدد فورًا.

!!! tip
    اضبط **Security** على `inbound_default` عندما تريد فقط تغيير العنوان/SNI مع الإبقاء على ما يتفاوض عليه inbound بالفعل.

## صيغ خرج اشتراك جديدة

**الموقع:** المستخدمون → تفاصيل المستخدم → **Subscription** (روابط لكل صيغة)، ونقطة النهاية العامة `/sub/{token}`

بالإضافة إلى الصيغ الموجودة `base64` و `clash` و `singbox`، تضيف v1.2.3 ثلاث صيغ قابلة للاختيار عبر معامل الاستعلام `?format=`:

| Format | `?format=` | الخرج |
|--------|-----------|--------|
| Xray / V2Ray JSON | `xray` | JSON خام لعميل Xray/V2Ray |
| Outline | `outline` | روابط Shadowsocks بصيغة `ss://` لـ Outline |
| روابط عادية | `links` | روابط مشاركة بنمط V2rayN، رابط في كل سطر |

المجموعة الكاملة الآن هي: `base64` و `clash` و `singbox` و `xray` و `outline` و `links`. عند عدم تحديد صيغة، تُكتشف الاستجابة تلقائيًا من User-Agent الخاص بالعميل.

```text
https://panel.example.com/sub/<token>?format=xray
https://panel.example.com/sub/<token>?format=outline
https://panel.example.com/sub/<token>?format=links
```

## حِزم قواعد التوجيه الذكية

**الموقع:** الشبكة والعُقد → حِزم التوجيه (Routing Packs)

**حزمة التوجيه** هي مجموعة مُسمّاة وقابلة لإعادة الاستخدام من قواعد التوجيه. أنشئها مرة واحدة، ثم طبّقها على أي عُقدة أو ضمّنها في اشتراكات Clash/sing-box — لا حاجة بعد الآن لإعادة إنشاء قواعد حظر الإعلانات / المباشر لإيران / البث لكل عُقدة.

### ما يمكنك فعله

| Action | الوصف |
|--------|-------------|
| إنشاء / تعديل | بناء حزمة من قواعد توجيه مرتّبة (نفس حقول توجيه العُقدة) |
| التطبيق على عُقدة | استبدال قواعد توجيه العُقدة بالحزمة وإعادة مزامنة هسته العُقدة |
| تعيين افتراضي عام | تُطبَّق حزمة واحدة على مستوى الأسطول بأكمله ما لم يتم تجاوزها |
| التعيين لكل مستخدم | امنح مستخدمًا معيّنًا حزمته الخاصة (مُضمَّنة في اشتراكه) |

### الإعداد

1. انقر على **New Pack** وسمِّها (مثل "إيران — حظر الإعلانات + مباشر")
2. أضف القواعد بترتيب الأولوية (نطاقات، عناوين IP، منافذ، وسوم inbound ← outbound/balancer)
3. احفظ، ثم:
   - **Apply** الحزمة على عُقدة لتفعيلها مباشرةً، أو
   - علّمها **Default** للأسطول بأكمله، أو
   - عيّنها لمستخدم من **المستخدمون ← مستخدم ← Routing Pack**

!!! note
    التعيين لكل مستخدم له الأولوية على الافتراضي العام. المستخدم الذي ليس له تعيين يعود إلى الحزمة الافتراضية.

## ماسح الـ IP النظيف (Clean-IP)

**الموقع:** الشبكة والعُقد → Clean IP

اعثر على عناوين IP لِحَوافّ Cloudflare/CDN ذات الأداء الجيد عبر فحص المرشحين وتقييمهم بناءً على زمن الاستجابة وفقدان الحزم. استخدم الأفضل منها كعنوان مُعلن في مضيف اشتراك أو سلسلة CDN.

### النتائج

| Field | الوصف |
|-------|-------------|
| IP | العنوان المرشح |
| Latency (ms) | زمن الذهاب والإياب |
| Loss % | فقدان الحزم المقاس |
| Score | الترتيب المُجمَّع (الأعلى أفضل) |
| Reachable | ما إذا اتصل الفحص |
| Scanned at | الطابع الزمني للقياس |

### الإعداد

1. الصق عناوين IP المرشحة في صندوق الفحص (عنوان في كل سطر)
2. اختياريًا اضبط **port** (الافتراضي `443`)
3. انقر على **Scan** — تُقيَّم النتائج وتُرتَّب من الأفضل إلى الأسوأ
4. انسخ أفضل عناوين IP إلى عنوان مضيف اشتراك أو سلسلة CDN/Relay

!!! warning "الحماية من SSRF"
    تُتحقَّق أهداف الفحص قبل الاتصال. تُرفض النطاقات الخاصة و loopback و link-local حتى لا يُستخدم الماسح للوصول إلى الخدمات الداخلية.

## فرض حد عناوين IP

**الموقع:** الأمان → IP Limit

افرض حدودًا متزامنة على عدد عناوين IP/الأجهزة لكل مستخدم للحدّ من مشاركة الحساب. عندما يتجاوز المستخدم حدّه، يُنفَّذ الإجراء المُعدّ.

### السياسة

| Setting | الوصف |
|---------|-------------|
| Enabled | تشغيل/إيقاف الفرض |
| Action | `warn` أو `disable_temporarily` أو `kill_connections` |
| Alert cooldown | الثواني بين التنبيهات المتكررة لنفس المستخدم |
| Restore after | الثواني قبل استعادة مستخدم مُعطَّل مؤقتًا |

### الإجراءات

| Action | السلوك |
|--------|----------|
| **warn** | تسجيل حدث وتنبيه؛ دون اتخاذ أي إجراء على الاتصال |
| **disable_temporarily** | تعطيل المستخدم لمدة `restore_after` ثانية، ثم استعادته |
| **kill_connections** | إسقاط الاتصالات المخالفة فورًا |

!!! note "اختلافات الهسته"
    `kill_connections` خاص بـ **Xray** فقط. على عُقد sing-box يتنزّل تلقائيًا إلى `disable_temporarily`، لأن sing-box لا يمكنه إسقاط اتصالات حيّة منفردة.

### الأحداث

يسرد جدول **Events** كل إجراء فرض: المستخدم، عدد عناوين IP المرصودة، عناوين IP نفسها، الإجراء المُتَّخَذ، والطابع الزمني.

## بروتوكولات ووسائط نقل جديدة ومصفوفة القدرات

**الموقع:** Inbounds → إضافة/تعديل (مُحدِّدات البروتوكول والنقل والأمان)

توسّع v1.2.3 تغطية البروتوكولات والنقل، وأصبح محرّر inbound الآن يقصر خياراتك على ما تدعمه هسته العُقدة المختارة فعليًا.

### Xray-core

- **بروتوكولات الإدخال:** `vless`، `vmess`، `trojan`، `shadowsocks` (+ SS-2022 متعدد المستخدمين)، `socks`، `http`، `dokodemo`
- **وسائط النقل:** `tcp`، `ws`، `grpc`، `httpupgrade`، `http`/`h2`، `xhttp`، `mkcp` (mKCP)
- **الأمان:** `none`، `tls`، `reality` — يأخذ TLS قائمة `alpn`؛ يدعم TCP نوع هدر `none`/`http`؛ يدعم xHTTP مُحدِّد `mode`

### sing-box

- **بروتوكولات الإدخال:** `vless`، `vmess`، `trojan`، `shadowsocks`، `hysteria2`، `tuic`، `wireguard`، `hysteria` (v1)، `shadowtls`، `anytls`، `socks`، `http`، `naive`
- **وسائط النقل:** `tcp`، `ws`، `grpc`، `httpupgrade`، `http`/`h2`، `quic`

### مصفوفة القدرات لكل بروتوكول

تكشف اللوحة **مصفوفة قدرات حيّة لكل بروتوكول** (`GET /api/capabilities`)، وهي **المصدر الوحيد للحقيقة** — لا يعرض محرّر inbound سوى تركيبات البروتوكول/النقل/الأمان التي تدعمها الهسته المختارة.

بعض البروتوكولات **لا تحمل نقلًا تدفقيًا** (stream transport):

| البروتوكول | الهسته | النقل | الأمان |
|----------|------|-----------|----------|
| `socks` | كلاهما | لا يوجد (TCP خام) | plaintext |
| `http` | كلاهما | لا يوجد (TCP خام) | plaintext |
| `naive` | sing-box | لا يوجد | **TLS إلزامي** |
| `dokodemo` | xray | لا يوجد (TCP/UDP خام) | plaintext |

!!! warning
    إدخالات `socks` و `http` تكون **plaintext** — لا تكشفها إلا على شبكات موثوقة أو خلف relay محلي. يفرض `naive` **TLS إلزاميًا**.

راجع [البروتوكولات](13-protocols-config.md) و `docs/protocols.md` للحصول على أمثلة إعداد كاملة.
