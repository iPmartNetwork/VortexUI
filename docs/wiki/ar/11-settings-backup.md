# الإعدادات والنسخ الاحتياطي

---

## إعدادات لوحة التحكم

صفحة **الإعدادات** تحتوي إعدادات لوحة التحكم العامة:

| الإعداد | الوصف |
|---------|-------|
| عنوان لوحة التحكم | يُعرض في تبويب المتصفح والرأس |
| رابط لوحة التحكم | الرابط العام (يُستخدم لروابط الاشتراك) |
| اللغة | لغة الواجهة الافتراضية |
| المنطقة الزمنية | المنطقة الزمنية للعرض |
| مدة JWT TTL | وقت انتهاء صلاحية التوكن |
| فاصل تحديث الاشتراك | عدد المرات التي يجب أن يُحدّث فيها العملاء |

---

## العلامة التجارية المخصصة

**الإعدادات → العلامة التجارية**

تخصيص مظهر لوحة التحكم:

| الحقل | الوصف |
|-------|-------|
| العنوان | عنوان لوحة التحكم (الرأس + تبويب المتصفح) |
| الشعار | رفع شعار مخصص (SVG/PNG) |
| أيقونة المتصفح | أيقونة متصفح مخصصة |
| لون التمييز | لون السمة الأساسي (`#hex`) |
| نص التذييل | سطر تذييل مخصص |
| خلفية تسجيل الدخول | خلفية مخصصة لصفحة تسجيل الدخول |

---

## العلامة التجارية الخاصة للموزّع (Whitelabel)

يمكن لكل موزّع تمييز بوابته بشكل مستقل:

**حساب الموزّع → العلامة التجارية**

| الحقل | الوصف |
|-------|-------|
| عنوان لوحة التحكم | عنوان لوحة/بوابة الموزّع |
| رابط الشعار | صورة العلامة التجارية للموزّع |
| لون التمييز | لون سمة `#hex` لمستخدميه |
| اسم البوابة | اسم URL مخصص اختياري |
| نص التذييل | سطر تذييل البوابة |

المستخدمون الذين يصلون للبوابة يرون العلامة التجارية لموزّعهم، وليس علامة لوحة التحكم الرئيسية.

## النسخ الاحتياطي والاستعادة

### نسخ احتياطي يدوي

**الإعدادات → النسخ الاحتياطي → إنشاء نسخة احتياطية**

ينشئ نسخة احتياطية كاملة تتضمن:

- تفريغ قاعدة البيانات (المستخدمون، العقد، الاتصالات الواردة، الخطط، الطلبات، الإعدادات)
- ملفات الإعداد
- شهادات TLS
- الأصول المرفوعة (الشعارات، إيصالات الدفع)

### نسخ احتياطي تلقائي

| الإعداد | الوصف |
|---------|-------|
| الجدول | تعبير cron (مثل `0 3 * * *` = يومياً الساعة 3 صباحاً) |
| الاحتفاظ | عدد النسخ الاحتياطية المحتفظ بها |
| تيليجرام | إرسال ملف النسخة لمحادثة المسؤول |
| S3 | الرفع لتخزين متوافق مع S3 |

### وجهات النسخ الاحتياطي

=== "تيليجرام"

    تُرسل النسخ الاحتياطية كملف لمعرّف محادثة المسؤول المُعدّ.
    أقصى حجم ملف: 50 ميغابايت (حدّ تيليجرام). النسخ الأكبر تُقسَّم.

=== "S3"

    إعداد تخزين متوافق مع S3:
    ```
    VORTEX_BACKUP_S3_BUCKET=my-backups
    VORTEX_BACKUP_S3_ENDPOINT=s3.amazonaws.com
    VORTEX_BACKUP_S3_ACCESS_KEY=xxxx
    VORTEX_BACKUP_S3_SECRET_KEY=xxxx
    VORTEX_BACKUP_S3_REGION=us-east-1
    ```

=== "محلي"

    تُخزَّن النسخ في `/var/lib/vortexui/backups/` (افتراضي).
    اضبط الاحتفاظ لمنع امتلاء القرص.

### الاستعادة

```bash
vortexui restore /path/to/backup.tar.gz
```

أو عبر الواجهة: **الإعدادات → النسخ الاحتياطي → استعادة → رفع ملف النسخة**.

!!! warning
    الاستعادة تكتب فوق قاعدة البيانات الحالية. أنشئ نسخة احتياطية جديدة قبل الاستعادة.

## إعداد الروابط العميقة

**الإعدادات → الروابط العميقة**

تكوين روابط الاشتراك العميقة لتكامل التطبيقات الأصلية:

| الإعداد | الوصف |
|---------|-------|
| الرابط الأساسي | الرابط العام للوحة التحكم |
| مخطط التطبيق | مخطط URL (مثل `vortex://`، `clash://`) |
| تضمين اسم الخادم | إضافة معرّف الخادم للروابط |
| شعار QR | شعار مخصص يُعرض في مركز رمز QR |

## فاحص التحديثات

**الإعدادات → التحديثات**

- التحقق من إصدارات VortexUI الجديدة
- عرض سجل التغييرات للتحديثات المتاحة
- تحديث بنقرة واحدة (يستدعي `vortexui update` داخلياً)
- تكوين فاصل التحقق التلقائي

## قوالب الإعدادات

**الإعدادات → قوالب الإعدادات**

تخصيص مخرجات إعدادات Clash/sing-box للاشتراكات:

### قالب Clash

تجاوز بنية Clash YAML الافتراضية:

- مجموعات بروكسي مخصصة (url-test، fallback، load-balance)
- قواعد مخصصة (حظر الإعلانات، حركة محلية مباشرة)
- إعدادات DNS
- قواعد توجيه مخصصة

### قالب sing-box

تجاوز بنية sing-box JSON الافتراضية:

- مجموعات اتصال صادر مخصصة
- قواعد التوجيه
- إعدادات DNS
- ميزات تجريبية

تدعم القوالب متغيرات:

| المتغير | يُوسَّع إلى |
|---------|------------|
| `{PROXIES}` | قائمة إعدادات البروكسي من اتصالات المستخدم الواردة |
| `{USERNAME}` | اسم المستخدم |
| `{SUB_URL}` | رابط الاشتراك |

## التدويل

تدعم لوحة التحكم **8 لغات** مع دعم RTL كامل:

- English (EN)
- فارسی (FA)
- Türkçe (TR)
- العربية (AR)
- Русский (RU)
- 中文 (ZH)
- 日本語 (JA)
- Español (ES)

بدّل اللغة من القائمة المنسدلة في الرأس. تُحفظ تفضيلات كل مسؤول بشكل مستقل.
