# مدیریت کاربران

!!! abstract "مدل کاربرمحور"
    یک کاربر = یک توکن سابسکریپشن = دسترسی به تمام اینباندهای اختصاص‌داده‌شده در همه نودها.
    نیازی به اکانت جداگانه برای هر پروتکل نیست.

---

## عملیات CRUD کاربر

### ایجاد کاربر

**کاربران → کاربر جدید**

| فیلد | توضیحات |
|------|---------|
| نام کاربری | شناسه یکتا |
| حد ترافیک | سقف ترافیک (بایت) — `0` = نامحدود |
| تاریخ انقضا | تاریخ انقضای اشتراک |
| حد دستگاه | حداکثر دستگاه‌های همزمان |
| استراتژی ریست | `none` / `daily` / `weekly` / `monthly` |
| وضعیت | `active` / `disabled` / `limited` |
| اینباندها | اینباندهای مجاز (یک یا چند مورد انتخاب کنید) |
| یادداشت | یادداشت فقط برای ادمین |

### عملیات گروهی

| عملیات | توضیحات |
|--------|---------|
| **ایجاد گروهی** | کاربران → افزودن گروهی — تعداد مشخص یا آپلود CSV |
| **انتخاب چندتایی** | چک‌باکس چند کاربر → اعمال عملیات |
| **اقدامات گروهی** | فعال، غیرفعال، حذف، تمدید، ریست ترافیک |
| **ایمپورت** | کاربران → ایمپورت — از دیتابیس 3x-ui یا Marzban |

## سهمیه‌ها و محدودیت‌ها

### حد ترافیک

سقف ترافیک به ازای هر کاربر تنظیم کنید. وقتی از حد بگذرد، وضعیت کاربر به `limited` تغییر کرده و رویداد `user.limited` فایر می‌شود.

### حد دستگاه

حداکثر اتصالات همزمان (IP‌های متمایز). اعمال محدودیت توسط [سیستم محدودیت IP](08-security-administration.md) انجام می‌شود.

### انقضا

کاربران در تاریخ تنظیم‌شده منقضی می‌شوند. سیستم رویداد `user.expiry_warning` را ۳ روز قبل و `user.expired` را در زمان انقضا فایر می‌کند.

### استراتژی ریست

| استراتژی | رفتار |
|----------|-------|
| `none` | هرگز ریست نمی‌شود — ترافیک برای همیشه تجمیع می‌شود |
| `daily` | ریست در نیمه‌شب UTC |
| `weekly` | ریست هر دوشنبه نیمه‌شب UTC |
| `monthly` | ریست در اول هر ماه نیمه‌شب UTC |

## تحویل سابسکریپشن

### لینک سابسکریپشن

هر کاربر یک URL سابسکریپشن یکتا دریافت می‌کند:

```
https://panel.example.com/sub/{token}
```

فرمت پاسخ از User-Agent کلاینت تشخیص داده می‌شود، یا با `?format=` اجبار می‌شود:

| فرمت | پارامتر | خروجی |
|------|---------|-------|
| Base64 | `?format=base64` | لینک‌های کدشده base64 سازگار با V2Ray |
| Clash | `?format=clash` | کانفیگ YAML کلش |
| sing-box | `?format=singbox` | کانفیگ JSON سینگ‌باکس |
| Xray JSON | `?format=xray` | JSON خام کلاینت Xray/V2Ray |
| Outline | `?format=outline` | لینک‌های `ss://` شدوساکس برای Outline |
| لینک‌های ساده | `?format=links` | لینک‌های اشتراک V2rayN، هر خط یکی |

### هاست‌های سابسکریپشن

بازنویسی‌های به ازای هر اینباند که آنچه سابسکریپشن تبلیغ می‌کند را تغییر می‌دهند — بدون دست‌زدن به کانفیگ فعال هسته.

| فیلد | توضیحات |
|------|---------|
| نام نمایشی | نام نمایش (از متغیرهای قالب پشتیبانی می‌کند) |
| آدرس | هاست/IP تبلیغ‌شده (مثلاً دامنه CDN) |
| پورت | بازنویسی پورت (`0` = ارث‌بری) |
| SNI | نام سرور TLS |
| Host | هدر HTTP `Host` |
| Path | مسیر WS/HTTPUpgrade/gRPC |
| ALPN | جداشده با کاما (مثلاً `h2,http/1.1`) |
| Fingerprint | فینگرپرینت uTLS (مثلاً `chrome`) |
| Security | `inbound_default`، `none`، `tls` یا `reality` |
| Allow insecure | رد تأیید گواهی |
| Mux | فعال‌سازی مالتی‌پلکس |
| Fragment | مشخصات فرگمنت TLS (مثلاً `1,40-60,30-50`) |
| Priority | ترتیب در اینباند (کمتر = زودتر رندر می‌شود) |

**متغیرهای قالب** — به ازای هر کاربر رندر می‌شوند:

| متغیر | تبدیل می‌شود به |
|--------|----------------|
| `{USERNAME}` | نام کاربری |
| `{SERVER_IP}` | آدرس نود |
| `{SERVER_PORT}` | پورت تبلیغ‌شده |
| `{PROTOCOL}` | پروتکل اینباند |
| `{NETWORK}` | نوع ترنسپورت |
| `{SECURITY}` | امنیت ترنسپورت |
| `{REMARK}` | نام نمایشی تنظیم‌شده |

!!! tip
    از هاست‌های سابسکریپشن برای قرار دادن یک اینباند پشت چندین دامنه CDN استفاده کنید، هر کدام با SNI و path متفاوت — همه از یک تعریف هاست با استفاده از متغیرهای قالب.

## پورتال سلف‌سرویس

**URL کاربر نهایی:** `/portal/login`

کاربران با توکن سابسکریپشن خود وارد شده و به موارد زیر دسترسی دارند:

| ویژگی | توضیحات |
|-------|---------|
| داشبورد | آمار مصرف، QR اشتراک، دستگاه/اتصال زنده، نمودار ۳۰ روزه، هشدار سهمیه، اقدامات سریع |
| پلن‌ها | مرور و خرید پلن اشتراک (از فروشگاه ریسلر خود) |
| تیکت‌ها | ارسال تیکت پشتیبانی، پاسخ به پیام‌های ادمین |
| رفرال | مشاهده/اشتراک کد رفرال، دیدن پاداش‌های کسب‌شده |

### داشبورد پورتال (نسخه ۱.۳.۱)

داشبورد در `/portal/dashboard` شامل موارد زیر است:

- **کارت اشتراک** — QR، کپی URL، لینک فرمت‌ها (Clash / sing-box / base64)
- **آمار زنده** — دستگاه فعال و اتصال live (هر ۱۵ ثانیه)
- **نمودار مصرف** — ۳۰ روز مصرف روزانه
- **هشدارها** — انقضا، نزدیک سقف (≥۹۰٪)، انقضای ۷ روز آینده
- **اقدامات سریع** — تمدید، تیکت، deep link (در صورت فعال بودن)

جزئیات API: [تغییرات نسخه‌ها](16-changelog.md).

### فروشگاه سلف‌سرویس

**URL:** `/sub/{token}/shop`

هر ریسلر پلن‌ها و روش‌های پرداخت خود را تنظیم می‌کند. کاربران نهایی فقط پیشنهادات ریسلر خود را می‌بینند:

1. کاربر پلن‌های موجود را مرور می‌کند
2. پلن و روش پرداخت را انتخاب می‌کند (زرین‌پال، کارت‌به‌کارت یا رمزارز)
3. پرداخت را تکمیل می‌کند (یا فیش آپلود می‌کند)
4. سفارش به‌صورت خودکار (زرین‌پال) یا پس از تأیید ادمین (کارت/رمزارز) تکمیل می‌شود

برای جزئیات کامل [پلن‌ها و پرداخت](09-plans-payments.md) را ببینید.

## اشتراک خانوادگی/گروهی

**کاربران → گروه‌های خانواده**

به کاربران اجازه دهید یک استخر داده مشترک داشته باشند:

1. یک **گروه خانواده** با حد ترافیک مشترک ایجاد کنید
2. کاربران موجود را به‌عنوان عضو اضافه کنید
3. ترافیک هر عضو از استخر مشترک کسر می‌شود
4. سقف اختیاری به ازای هر عضو درون استخر مشترک

| فیلد | توضیحات |
|------|---------|
| نام | نام گروه |
| مالک | اکانت اصلی |
| حد ترافیک | کل استخر مشترک |
| حداکثر اعضا | محدودیت (پیش‌فرض: ۵) |
| سهمیه عضو | سقف به ازای هر عضو درون استخر |

## سهمیه هوشمند

کاهش تدریجی سرعت به‌جای قطع سخت کاربران در حد مصرف‌شان.

سطوح را به‌صورت JSON تنظیم کنید:

```json
[
  { "threshold_pct": 80, "action": "warn", "speed_limit": 0 },
  { "threshold_pct": 95, "action": "throttle", "speed_limit": 524288 },
  { "threshold_pct": 100, "action": "disable" }
]
```

- در **۸۰%** → به کاربر هشدار بده (رویداد اعلان)
- در **۹۵%** → محدودسازی سرعت به ۵۱۲ KB/s
- در **۱۰۰%** → غیرفعال‌سازی اکانت

!!! info
    سهمیه هوشمند به ازای هر پلن یا به‌صورت سراسری قابل تنظیم است. تنظیمات پلن بر پیش‌فرض سراسری اولویت دارد.

## سیستم رفرال

**کاربران → رفرال‌ها**

| تنظیم | توضیحات | پیش‌فرض |
|--------|---------|---------|
| فعال | روشن/خاموش کردن رفرال | خاموش |
| نوع پاداش | `data` (گیگابایت اضافه) یا `days` (زمان اضافه) | `data` |
| مقدار پاداش | چقدر به ازای هر رفرال | ۱ گیگابایت |
| حداکثر رفرال | محدودیت به ازای هر کاربر (`0` = نامحدود) | `0` |
| نیاز به پرداخت | فقط برای رفرال‌های پرداخت‌کننده پاداش بده | خاموش |
| پاداش دوطرفه | پاداش به دعوت‌کننده + کاربر جدید | بله |

کاربران از طریق پورتال به کد رفرال خود دسترسی دارند. وقتی یک دوست با آن کد ثبت‌نام کند، هر دو طرف پاداش دریافت می‌کنند.

## قالب‌های کانفیگ

خروجی Clash/sing-box را که کاربران در سابسکریپشن دریافت می‌کنند سفارشی کنید:

- قوانین مسیریابی سفارشی اضافه کنید (مسدودسازی تبلیغات، ترافیک محلی مستقیم)
- تنظیمات DNS را پیکربندی کنید
- استراتژی‌های گروه پروکسی تنظیم کنید (url-test, fallback, load-balance)
- قالب‌های به ازای هر کاربر یا سراسری

## دیپ‌لینک و کد QR

**سیستم → دیپ‌لینک‌ها**

دیپ‌لینک‌های سابسکریپشن برای راه‌اندازی اپ با یک لمس تولید کنید:

| تنظیم | توضیحات |
|--------|---------|
| Base URL | URL عمومی پنل |
| App scheme | URL scheme برای اپ‌های بومی |
| شامل نام | افزودن نام سرور به لینک |
| لوگو QR | لوگو سفارشی در مرکز کد QR |

## ایمپورت از سایر پنل‌ها

**کاربران → ایمپورت**

مهاجرت کاربران از:

- **3x-ui** — مسیر فایل دیتابیس را ارائه دهید
- **Marzban** — رشته اتصال دیتابیس یا فایل export

ایمپورتر کاربران را نگاشت می‌کند و نام‌های کاربری، حدود ترافیک و تاریخ‌های انقضا را حفظ می‌کند.
