# اعلان‌ها

!!! info "چند کاناله"
    VortexUI از وب‌هوک، تلگرام و اعلان‌های درون‌برنامه‌ای پشتیبانی می‌کند. هر کانال
    به‌صورت مستقل قابل تنظیم بوده و می‌تواند زیرمجموعه‌های متفاوتی از رویدادها دریافت کند.

---

## وب‌هوک (HMAC-SHA256)

**تنظیمات → اعلان‌ها → وب‌هوک**

دریافت پیلودهای JSON ساختاریافته برای رویدادهای پنل در اندپوینت شما.

### پیکربندی

| تنظیم | توضیحات |
|--------|---------|
| URL | اندپوینت وب‌هوک شما |
| Secret | کلید امضای HMAC-SHA256 |
| رویدادها | انتخاب رویدادهای ارسالی |
| فعال | کلید روشن/خاموش |

### تأیید اعتبار

هر درخواست شامل هدر `X-Signature-256` است:

```
X-Signature-256: sha256=<hex_hmac>
```

با محاسبه `HMAC-SHA256(request_body, secret)` و مقایسه تأیید کنید.

### ساختار پیلود رویداد

```json
{
  "event": "user.limited",
  "timestamp": "2025-01-15T10:30:00Z",
  "data": {
    "user_id": "uuid",
    "username": "testuser",
    "data_limit": 53687091200,
    "used_traffic": 53687091200
  }
}
```

### رویدادهای موجود

| رویداد | محرک |
|--------|------|
| `user.created` | کاربر جدید ایجاد شد |
| `user.deleted` | کاربر حذف شد |
| `user.limited` | کاربر به حد ترافیک رسید |
| `user.expired` | اشتراک کاربر منقضی شد |
| `user.expiry_warning` | ۳ روز تا انقضا |
| `user.enabled` | کاربر مجدداً فعال شد |
| `user.disabled` | کاربر به‌صورت دستی غیرفعال شد |
| `node.offline` | نود اتصال را از دست داد |
| `node.online` | نود مجدداً متصل شد |
| `node.unhealthy` | بررسی سلامت نود ناموفق |
| `order.created` | سفارش خرید جدید |
| `order.paid` | پرداخت سفارش تأیید شد |
| `backup.completed` | بکاپ تکمیل شد |
| `admin.login` | رویداد ورود ادمین |

## بات تلگرام

**تنظیمات → اعلان‌ها → تلگرام**

### بات ادمین

اعلان‌ها را به چت تلگرام ادمین ارسال می‌کند:

| تنظیم | توضیحات |
|--------|---------|
| توکن بات | از [@BotFather](https://t.me/BotFather) |
| شناسه چت ادمین | شناسه چت شخصی یا گروهی شما |
| رویدادها | انتخاب رویدادهای اعلان |

### بات سمت کاربر

کاربران می‌توانند با بات از طریق توکن سابسکریپشن تعامل کنند:

| دستور | عملکرد |
|--------|--------|
| `/start` | لینک اکانت با توکن اشتراک |
| `/usage` | مشاهده مصرف فعلی |
| `/renew` | دریافت لینک تمدید |
| `/status` | بررسی وضعیت اکانت |
| `/help` | لیست دستورات موجود |

### قالب‌های اعلان

فرمت پیام را با متغیرهای قالب سفارشی کنید:

```
🔔 کاربر محدود شد
نام کاربری: {username}
مصرف‌شده: {used_traffic}
حد: {data_limit}
```

## اعلان‌های سهمیه

**تنظیمات → اعلان‌ها → هشدارهای سهمیه**

هنگام نزدیک‌شدن کاربران به حد ترافیک خود هشدار دهید:

| تنظیم | توضیحات |
|--------|---------|
| فعال | فعال‌سازی اعلان سهمیه |
| آستانه‌ها | درصدهای فعال‌سازی (مثلاً ۸۰%، ۹۰%، ۱۰۰%) |
| تلگرام | ارسال از طریق بات |
| وب‌هوک | ارسال به URL وب‌هوک |
| فاصله زمانی | دقیقه بین هشدارهای تکراری |

---

## هشدارهای سهمیه ریسلر

**نوار کناری → هشدارهای سهمیه ریسلر** (فقط ادمین sudo)

مانیتور کنید وقتی ریسلرها به حد استخر ترافیک/کاربر خود نزدیک می‌شوند:

| تنظیم | توضیحات |
|--------|---------|
| فعال | کلید سراسری |
| تلگرام | ارسال به بات پنل |
| URL وب‌هوک | اندپوینت خارجی اختیاری |
| آستانه‌ها | درصدها (مثلاً ۸۰، ۹۰، ۱۰۰) |
| فاصله زمانی | دقیقه بین هشدارها |
| هشدارهای اخیر | جدول هشدارهای فعال‌شده |

## مرکز اعلان‌ها (دراپ‌داون زنگ)

آیکون زنگ در هدر پنل اعلان‌های اخیر را نمایش می‌دهد:

- نشانگر تعداد خوانده‌نشده
- کلیک برای باز کردن دراپ‌داون
- هر اعلان نمایش می‌دهد: نوع رویداد، توضیحات، زمان
- کلیک برای رفتن به منبع مربوطه
- علامت‌گذاری خوانده‌شده / علامت‌گذاری همه خوانده‌شده

به ازای هر اکانت ادمین ذخیره می‌شود.

## رویدادهای زنده SSE

پنل از Server-Sent Events برای بروزرسانی بلادرنگ UI استفاده می‌کند:

| استریم | محتوا |
|--------|-------|
| `/api/sse/events` | رویدادهای سیستم (وضعیت نود، محدودیت کاربر و غیره) |
| `/api/sse/stats` | آمار زنده (شمارنده اتصالات، ترافیک) |
| `/api/sse/monitor` | بروزرسانی اتصالات فعال |

کامپوننت‌های فرانتند به‌صورت خودکار subscribe می‌شوند. نیازی به تنظیم نیست — SSE همیشه فعال است.

!!! tip
    رویدادهای SSE نمودارهای بلادرنگ داشبورد، صفحه مانیتور و زنگ اعلان را تغذیه می‌کنند.
    اگر پروکسی معکوس شما پاسخ‌ها را بافر می‌کند، مطمئن شوید استریم SSE مجاز است (Caddy به‌صورت پیش‌فرض این کار را انجام می‌دهد).
