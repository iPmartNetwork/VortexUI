# گروه‌های پروتکل

!!! abstract "سوئیچ خودکار پروتکل"
    اینباندها را در یک گروه منطقی دسته‌بندی کنید تا کلاینت‌ها به‌صورت خودکار بین پروتکل‌ها جابجا شوند.
    هر گروه به‌صورت بلوک `urltest` در سابسکریپشن رندر می‌شود.

---

## مرور کلی

**گروه‌های پروتکل** امکان دسته‌بندی چند اینباند در یک واحد منطقی و ارائه آن‌ها به‌عنوان یک بلوک سابسکریپشن را فراهم می‌کند. به‌جای نمایش تک‌تک اینباندها، کاربران مجموعه‌ای سازمان‌یافته و اولویت‌بندی‌شده از اتصالات مناسب ISP، منطقه یا کاربردشان دریافت می‌کنند.

!!! note
    گروه‌های پروتکل به VortexUI v1.4.0+ با نودهای sing-box نیاز دارند. نودهای Xray لیست ساده (fallback) دریافت می‌کنند.

---

## چرا گروه‌های پروتکل؟

| مشکل | راه‌حل |
|------|--------|
| کاربران به پروتکل‌های غیربهینه متصل می‌شوند | گروه‌ها خودکار بر اساس امتیاز کیفیت مرتب می‌کنند |
| ISP یک پروتکل را مسدود کند → همه کاربران قطع شوند | گروه‌ها بدون وقفه به اینباند بعدی می‌روند |
| ادمین‌ها نمی‌توانند پشته‌های پروتکل را A/B تست کنند | گروه‌ها پروفایل‌های ISP مجزا ارائه می‌دهند |
| شلوغی سابسکریپشن (۲۰+ کانفیگ) | گروه‌ها در بلوک‌های `urltest` جمع می‌شوند |

---

## ایجاد گروه پروتکل

### از طریق UI

1. به **اینباندها → تب گروه‌های پروتکل** بروید
2. روی **+ گروه جدید** کلیک کنید
3. نام‌گذاری کنید (مثلاً «بهینه MCI ایران»)
4. اینباندها را از پنل سمت چپ به داخل گروه بکشید
5. ترتیب اولویت را تنظیم کنید (بالا = بالاترین اولویت)
6. ذخیره کنید

### از طریق API

```bash
curl -X POST https://panel.example.com/api/protocol-groups \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Iran MCI Optimized",
    "inbound_ids": [12, 7, 3, 19],
    "isp_profile": "MCI",
    "enabled": true
  }'
```

**پاسخ:**

```json
{
  "id": 5,
  "name": "Iran MCI Optimized",
  "inbound_ids": [12, 7, 3, 19],
  "isp_profile": "MCI",
  "quality_score": 87,
  "created_at": "2025-01-15T10:30:00Z"
}
```

## پروفایل‌های ISP

هر گروه می‌تواند یک ISP خاص را هدف قرار دهد. هنگام درخواست سابسکریپشن، VortexUI از طریق MaxMind GeoIP اپراتور کاربر را شناسایی و گروه مناسب را ارائه می‌کند.

| کد ISP | هدف | رفتار |
|--------|-----|-------|
| `MCI` | همراه اول | ضد DPI تهاجمی، ترجیح Reality |
| `IRANCELL` | ایرانسل MTN | فیلتر متوسط، WS+TLS کار می‌کند |
| `TCI` | مخابرات ایران | فیلتر سنگین، gRPC+Reality الزامی |
| `SHATEL` | شاتل | متوسط، TLS استاندارد کافی است |
| `DEFAULT` | سایر / ناشناخته | ترکیب متعادل پروتکل‌ها |

تنظیم پروفایل ISP:

```bash
curl -X PATCH https://panel.example.com/api/protocol-groups/5 \
  -H "Authorization: Bearer $TOKEN" \
  -d '{ "isp_profile": "MCI" }'
```

## تغییر ترتیب اینباندها

ترتیب اولویت تعیین می‌کند کلاینت ابتدا کدام اینباند را در بلوک `urltest` امتحان کند.

### کشیدن و رها کردن (UI)

در **پنل گروه‌های پروتکل**، چیپ‌های اینباند را بکشید و ترتیب دهید. تغییرات خودکار ذخیره می‌شوند.

### از طریق API

```bash
curl -X POST https://panel.example.com/api/protocol-groups/5/reorder \
  -H "Authorization: Bearer $TOKEN" \
  -d '{ "inbound_ids": [19, 12, 7, 3] }'
```

## رندر سابسکریپشن

هنگام دریافت `/sub/{token}` توسط کلاینت، VortexUI گروه‌های پروتکل را به‌صورت بلوک‌های `urltest` رندر می‌کند:

```json
{
  "outbounds": [
    {
      "type": "urltest",
      "tag": "Iran-MCI-Optimized",
      "outbounds": ["vless-reality-mci", "trojan-ws-cdn", "vmess-grpc-relay"],
      "url": "https://cp.cloudflare.com/generate_204",
      "interval": "3m",
      "tolerance": 100
    }
  ]
}
```

کاربران Clash Meta معادل `proxy-group` با استراتژی `url-test` دریافت می‌کنند.

## سوئیچ خودکار پروتکل

گروه‌های پروتکل از **جابجایی خودکار** پشتیبانی می‌کنند — وقتی کیفیت پروتکل فعال افت کند، کلاینت بدون وقفه به اینباند بعدی failover می‌کند.

### رویدادهای سوئیچ

مانیتور رویدادهای سوئیچ از طریق API:

```bash
curl -X POST https://panel.example.com/sub/{token}/switch \
  -H "Content-Type: application/json" \
  -d '{
    "from_inbound": 12,
    "to_inbound": 7,
    "reason": "timeout",
    "latency_ms": 3200
  }'
```

رویدادهای سوئیچ به موتور ترتیب‌دهی تطبیقی ارسال می‌شوند.

### ترتیب‌دهی تطبیقی

VortexUI تلمتری رویدادهای سوئیچ را جمع‌آوری کرده و ترتیب گروه را **خودترمیم** می‌کند:

1. جمع‌آوری رویدادهای سوئیچ هر گروه در پنجره ۲۴ ساعته
2. محاسبه نرخ موفقیت هر اینباند (اتصال‌ها / timeout‌ها)
3. مرتب‌سازی مجدد: بالاترین نرخ موفقیت → جایگاه ۱
4. اعمال ترتیب جدید به رندرهای آینده سابسکریپشن

!!! note
    ترتیب‌دهی تطبیقی هر ۶ ساعت اجرا می‌شود. تغییر دستی ترتیب تا چرخه بعدی اولویت دارد.

## رابط کاربری

**پنل گروه‌های پروتکل** از صفحه اینباندها در ساید‌بار قابل دسترسی است:

| عنصر | توضیحات |
|------|---------|
| لیست گروه‌ها | تمام گروه‌ها با نشان وضعیت و امتیاز کیفیت |
| مخزن اینباند | اینباندهای موجود (بکشید برای افزودن) |
| ناحیه ترتیب | کشیدن چیپ‌ها برای تعیین اولویت |
| انتخاب‌گر ISP | دراپ‌داون برای اختصاص پروفایل ISP |
| کارت آمار | رویدادهای سوئیچ ۲۴ ساعته، میانگین تأخیر، نرخ موفقیت |
| دکمه همگام‌سازی | اجبار رندر مجدد تمام سابسکریپشن‌ها با ترتیب جدید |

---

## بهترین روش‌ها

1. **یک گروه به ازای هر ISP** — از پروفایل‌های همپوشان اجتناب کنید
2. **۳ تا ۵ اینباند در هر گروه** — تعداد زیاد باعث افزایش حجم سابسکریپشن و سربار urltest می‌شود
3. **ترکیب انواع ترانسپورت** — Reality + WS + gRPC را ترکیب کنید برای تاب‌آوری
4. **مانیتور رویدادهای سوئیچ** — سوئیچ‌های مکرر نشان‌دهنده مشکل یک اینباند است
5. **اجازه تثبیت ترتیب تطبیقی** — در ۴۸ ساعت اول گروه جدید دستی تغییر ندهید
