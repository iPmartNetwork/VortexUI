# تنظیمات و بکاپ

---

## تنظیمات پنل

صفحه **تنظیمات** شامل پیکربندی عمومی پنل است:

| تنظیم | توضیحات |
|--------|---------|
| عنوان پنل | نمایش در تب مرورگر و هدر |
| URL پنل | آدرس عمومی (برای لینک‌های سابسکریپشن) |
| زبان | زبان پیش‌فرض رابط |
| منطقه زمانی | منطقه زمانی پنل برای نمایش |
| TTL JWT | زمان انقضای توکن |
| بازه بروزرسانی سابسکریپشن | هر چند وقت کلاینت‌ها باید رفرش کنند |

---

## برندسازی سفارشی

**تنظیمات → برندسازی**

ظاهر پنل را سفارشی کنید:

| فیلد | توضیحات |
|------|---------|
| عنوان | عنوان پنل (هدر + تب مرورگر) |
| لوگو | آپلود لوگو سفارشی (SVG/PNG) |
| Favicon | فاوآیکون سفارشی مرورگر |
| رنگ تم | رنگ اصلی تم (`#hex`) |
| متن فوتر | خط سفارشی فوتر |
| پس‌زمینه ورود | پس‌زمینه سفارشی صفحه ورود |

---

## وایت‌لیبل ریسلر

هر ریسلر می‌تواند پورتال خود را به‌صورت مستقل برندسازی کند:

**اکانت ریسلر → برندسازی**

| فیلد | توضیحات |
|------|---------|
| عنوان پنل | عنوان پنل/پورتال ریسلر |
| URL لوگو | تصویر برند ریسلر |
| رنگ تم | رنگ تم `#hex` برای کاربرانش |
| اسلاگ پورتال | اسلاگ URL سفارشی اختیاری |
| متن فوتر | خط فوتر پورتال |

کاربرانی که به پورتال دسترسی پیدا می‌کنند، برندسازی ریسلر خود را می‌بینند نه پنل اصلی.

## بکاپ و بازیابی

### بکاپ دستی

**تنظیمات → بکاپ → ایجاد بکاپ**

یک بکاپ کامل شامل:

- دامپ دیتابیس (کاربران، نودها، اینباندها، پلن‌ها، سفارشات، تنظیمات)
- فایل‌های پیکربندی
- گواهی‌های TLS
- فایل‌های آپلودشده (لوگوها، فیش‌های پرداخت)

### بکاپ خودکار

| تنظیم | توضیحات |
|--------|---------|
| زمان‌بندی | عبارت cron (مثلاً `0 3 * * *` = روزانه ساعت ۳ صبح) |
| نگهداری | تعداد بکاپ‌های قابل نگهداری |
| تلگرام | ارسال فایل بکاپ به چت ادمین |
| S3 | آپلود به استوریج سازگار S3 |

### مقاصد بکاپ

=== "تلگرام"

    بکاپ‌ها به‌صورت فایل به شناسه چت ادمین تنظیم‌شده ارسال می‌شوند.
    حداکثر حجم فایل: ۵۰ مگابایت (محدودیت تلگرام). بکاپ‌های بزرگ‌تر تقسیم می‌شوند.

=== "S3"

    استوریج سازگار S3 را تنظیم کنید:
    ```
    VORTEX_BACKUP_S3_BUCKET=my-backups
    VORTEX_BACKUP_S3_ENDPOINT=s3.amazonaws.com
    VORTEX_BACKUP_S3_ACCESS_KEY=xxxx
    VORTEX_BACKUP_S3_SECRET_KEY=xxxx
    VORTEX_BACKUP_S3_REGION=us-east-1
    ```

=== "محلی"

    بکاپ‌ها در `/var/lib/vortexui/backups/` (پیش‌فرض) ذخیره می‌شوند.
    نگهداری را تنظیم کنید تا دیسک پر نشود.

### بازیابی

```bash
vortexui restore /path/to/backup.tar.gz
```

یا از طریق UI: **تنظیمات → بکاپ → بازیابی → آپلود فایل بکاپ**.

!!! warning
    بازیابی دیتابیس فعلی را بازنویسی می‌کند. قبل از بازیابی یک بکاپ جدید بگیرید.

## تنظیم دیپ‌لینک

**تنظیمات → دیپ‌لینک‌ها**

تنظیم دیپ‌لینک‌های سابسکریپشن برای یکپارچگی اپ بومی:

| تنظیم | توضیحات |
|--------|---------|
| Base URL | آدرس عمومی پنل |
| App scheme | URL scheme (مثلاً `vortex://`، `clash://`) |
| شامل نام سرور | افزودن شناسه سرور به لینک‌ها |
| لوگو QR | لوگو سفارشی در مرکز کد QR |

---

## بررسی بروزرسانی

**تنظیمات → بروزرسانی‌ها**

- بررسی نسخه‌های جدید VortexUI
- مشاهده تاریخچه تغییرات برای بروزرسانی‌های موجود
- بروزرسانی با یک کلیک (داخلی `vortexui update` را اجرا می‌کند)
- تنظیم بازه بررسی خودکار

## قالب‌های کانفیگ

**تنظیمات → قالب‌های کانفیگ**

سفارشی‌سازی خروجی کانفیگ Clash/sing-box برای سابسکریپشن‌ها:

### قالب Clash

بازنویسی ساختار پیش‌فرض YAML کلش:

- گروه‌های پروکسی سفارشی (url-test, fallback, load-balance)
- قوانین سفارشی (مسدودسازی تبلیغات، ترافیک محلی مستقیم)
- تنظیمات DNS
- قوانین مسیریابی سفارشی

### قالب sing-box

بازنویسی ساختار پیش‌فرض JSON سینگ‌باکس:

- گروه‌های اوتباند سفارشی
- قوانین مسیر
- تنظیمات DNS
- ویژگی‌های آزمایشی

قالب‌ها از متغیرها پشتیبانی می‌کنند:

| متغیر | تبدیل می‌شود به |
|--------|----------------|
| `{PROXIES}` | لیست کانفیگ‌های پروکسی از اینباندهای کاربر |
| `{USERNAME}` | نام کاربری |
| `{SUB_URL}` | URL سابسکریپشن |

## بین‌المللی‌سازی

پنل از **۸ زبان** با پشتیبانی کامل RTL پشتیبانی می‌کند:

- English (EN)
- فارسی (FA)
- Türkçe (TR)
- العربية (AR)
- Русский (RU)
- 中文 (ZH)
- 日本語 (JA)
- Español (ES)

از دراپ‌داون هدر زبان را تغییر دهید. ترجیح هر ادمین به‌صورت مستقل ذخیره می‌شود.
